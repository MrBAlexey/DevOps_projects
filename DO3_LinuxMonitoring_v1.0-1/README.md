# Linux Monitoring v1.0 (Linux basic bash scripting and system research)

Проект показывает, как с помощью **Bash** и стандартных утилит Linux:

- собирать ключевую информацию о системе (хостнейм,TimeZone, пользователь, ОС, время работы, IP, маска, шлюз, память, диск);
- валидировать входные параметры скриптов (тип, количество, формат);
- выводить информацию в **цветном, настраиваемом виде** с помощью ANSI-кодов;
- сохранять результаты в **файл с временной меткой**;
- анализировать **файловую систему**:
  - считать количество папок и файлов;
  - считать файлы по типам (`.conf`, `.txt`, `.log`, архивы, исполняемые, символические ссылки);
  - находить **топ-5 самых больших папок** и **топ-10 самых больших файлов**;
  - находить **топ-10 самых больших исполняемых файлов** с вычислением их **MD5-хеша**;
  - измерять **время выполнения скрипта**.

***

## Что реализовано в проекте (по частям)

### Part 1. Проба пера

- Написан простой Bash-скрипт:
  - принимает **один текстовый параметр**;
  - выводит этот параметр;
  - если параметр — число, выводит сообщение о **некорректном вводе**.
- Реализована базовая проверка ввода через шаблонное сопоставление в Bash.

### Part 2. Исследование системы

- Написан скрипт, который собирает и выводит информацию о системе:

  - `HOSTNAME` — сетевое имя хоста;
  - `TIMEZONE` — временная зона в виде `America/New_York UTC -5`;
  - `USER` — текущий пользователь;
  - `OS` — тип и версия ОС (из `/etc/os-release`);
  - `DATE` — текущая дата и время в формате `12 May 2020 12:24:36`;
  - `UPTIME` — время работы системы (человекочитаемый формат);
  - `UPTIME_SEC` — время работы в секундах;
  - `IP` — IP-адрес машины (любой интерфейс);
  - `MASK` — сетевая маска в виде `xxx.xxx.xxx.xxx`;
  - `GATEWAY` — IP шлюза по умолчанию;
  - `RAM_TOTAL` — общая память в Гб (3 знака после запятой);
  - `RAM_USED` — занятая память в Гб;
  - `RAM_FREE` — свободная память в Гб;
  - `SPACE_ROOT` — размер корневого раздела в МБ (2 знака после запятой);
  - `SPACE_ROOT_USED` — занятое пространство в МБ;
  - `SPACE_ROOT_FREE` — свободное пространство в МБ.

- После вывода данных скрипт предлагает пользователю сохранить информацию в файл (**Y/N**):
  - при ответе `Y`/`y` создаётся файл с именем вида `DD_MM_YY_HH_MM_SS.status`;
  - в файл записываются те же данные, что были выведены на экран.

### Part 3. Визуальное оформление вывода для скрипта исследования системы

- На базе скрипта из **Part 2** создан новый скрипт:
  - **без сохранения в файл**;
  - с поддержкой **цветного вывода**.
- Скрипт принимает **4 числовых параметра** (от 1 до 6):
  - параметр 1 — фон названий значений (`HOSTNAME`, `TIMEZONE`, `USER` и т.д.);
  - параметр 2 — цвет шрифта названий;
  - параметр 3 — фон самих значений (после `=`);
  - параметр 4 — цвет шрифта значений.
- Цвета:
  - 1 — white  
  - 2 — red  
  - 3 — green  
  - 4 — blue  
  - 5 — purple  
  - 6 — black

- Реализована проверка:
  - если фон и цвет шрифта в каком-то столбце совпадают — выводится сообщение об ошибке и скрипт завершается;
  - при некорректных параметрах (вне диапазона 1–6 или не 4 параметра) — выводится сообщение об ошибке и скрипт завершается.

### Part 4. Конфигурирование визуального оформления вывода

- На базе скрипта из **Part 3** создан скрипт:
  - который **запускается без параметров**;
  - цвета задаются в **конфигурационном файле** `config.cfg`.

Формат конфига:
```ini
column1_background=2
column1_font_color=4
column2_background=5
column2_font_color=1
```

- Если какой-то параметр не задан в конфиге, используется **цветовая схема по умолчанию** (на выбор разработчика).
- После вывода информации о системе скрипт выводит применённую цветовую схему:

```text
Column 1 background = 2 (red)
Column 1 font color = 4 (blue)
Column 2 background = 5 (purple)
Column 2 font color = 1 (white)
```

При использовании схемы по умолчанию:

```text
Column 1 background = default (black)
Column 1 font color = default (white)
Column 2 background = default (red)
Column 2 font color = default (blue)
```

### Part 5. Исследование файловой системы

- Написан скрипт, который принимает **один параметр**:
  - путь к директории (абсолютный или относительный), обязательно заканчивающийся на `/`, например: `script05.sh /var/log/`.

Скрипт выводит:

- Общее число папок (включая вложенные);
- **TOP-5 папок** с самым большим весом (путь и размер);
- Общее число файлов;
- Количество файлов по типам:
  - конфигурационные (`.conf`);
  - текстовые (`.txt`, `.md`);
  - исполняемые;
  - лог-файлы (`.log`);
  - архивы (`.tar`, `.gz`, `.bz2`, `.xz`, `.zip`, `.7z`, `.rar`);
  - символические ссылки.
- **TOP-10 файлов** с самым большим весом (путь, размер, тип по расширению);
- **TOP-10 исполняемых файлов** с самым большим весом (путь, размер, MD5-хеш);
- **Время выполнения скрипта** в секундах.

Вывод строго оформлен в заданном формате, как в условии проекта.

## Ключевые технологии и инструменты

- Операционная система: **Ubuntu Server 20.04 LTS**
- Язык: **Bash**  
- Оболочка: **Bash (Bourne-Again SHell)**  
- Утилиты:
  - `hostname`, `timedatectl`, `whoami`, `date`, `uptime`, `free`, `df`, `ip`, `awk`, `find`, `du`, `sort`, `head`, `md5sum`, `numfmt`, `bc`
- Файлы:
  - `/etc/os-release`, `/proc/uptime`, `/etc/ssh/sshd_config` (косвенно), `config.cfg`
- ANSI-коды:
  - для цветов фона и шрифта в терминале (`\e[...m`, `\e[0m`)

## Структура проекта

- Все скрипты находятся в папке `src/`.
- Для каждого скрипта (Part) создана папка вида `01`, `02`, `03`, `04`, `05`.
- Основной скрипт в каждой части называется `main.sh`.
- Вспомогательные скрипты Декомпозированы:
  - `get_data.sh` — сбор данных о системе;
  - `print_text.sh` — цветной вывод текста;
  - `color_selection.sh` — выбор цветов из конфига;
  - `get_dir_information.sh` — сбор данных о файловой системе;
  - `print_data.sh` — вывод данных о директории;
  - `texting_param.sh` — обработка параметра в Part 1.
- Все скрипты имеют **проверку на некорректный ввод**:
  - missing parameters;
  - неправильный тип/формат;
  - несуществующая директория и т.п.

Структура проекта:

```text
src/
  01/
    main.sh
    texting_param.sh
  02/
    main.sh
    get_data.sh
  03/
    main.sh
    get_data.sh
    print_text.sh
  04/
    main.sh
    config.cfg
    color_selection.sh
    get_data.sh
    print_text.sh
  05/
    main.sh
    print_data.sh
    get_dir_information.sh
```

## Как запустить

1. На **Ubuntu Server 20.04 LTS**.
2. Перейди в папку с проектом:
   ```bash
   cd path/to/Linux_Monitoring/src
   ```
3. Для каждого задания выполни:

   - **Part 1**:
     ```bash
     ./01/main.sh "some text"
     ```
   - **Part 2**:
     ```bash
     ./02/main.sh
     ```
   - **Part 3**:
     ```bash
     ./03/main.sh 1 3 4 5
     ```
   - **Part 4**:
     - Отредактируй `04/config.cfg` при желании:
       ```ini
       column1_background=3
       column1_font_color=2
       column2_background=5
       column2_font_color=1
       ```
     - Запусти:
       ```bash
       ./04/main.sh
       ```
   - **Part 5**:
     ```bash
     ./05/main.sh /var/log/
     ```
